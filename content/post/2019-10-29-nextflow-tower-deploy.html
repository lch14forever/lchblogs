测试环境

Ubuntu 18.04 （Root权限）

安装jdk（>1.8）及编译依赖

sudo apt update
sudo apt install openjdk-8-jdk-headless
sudo apt install build-essential

安装docker，docker-compose

sudo apt install docker.io
sudo curl -L "https://github.com/docker/compose/releases/download/1.24.1/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

更改docker权限（此处需要登出并重新登陆）

sudo usermod -a -G docker $USER

下载nf-tower

## install git
sudo apt install git
git clone https://github.com/seqeralabs/nf-tower

配置环境变量

此处需要申请一个发邮件的服务器，nf推荐了https://sendgrid.com

在左侧选项栏点击Settings–>APIKeys–>Create API Key–>输入key name并选择Full Access 点击生成的秘钥直接拷贝到剪切板，如“SG.dsVDEPp9SCOJFxe8QObVcAxxxxxxxxxxxxx”

以下环境变量写入~/.bashrc（Ubuntu下也可用~/.bash_aliases）

export TOWER_SMTP_HOST="smtp.sendgrid.net"
export TOWER_SMTP_PORT="587"
export TOWER_SMTP_USER="apikey"
export TOWER_SMTP_PASSWORD="SG.dsVDEPp9SCOJFxe8QObVcAxxxxxxxxxxxxx"

编译tower

make build
make run

使用Nextflow Tower

使用方法基本与tower.nf网站一致。有一点需要注意，如果nextflow部署在服务器上，需要修改邮件中的链接地址：

  • 邮件中链接为：http://localhost:8000/auth?email=用户名%40邮箱域名&authToken=c5618b821d6ccxxxxxxxxxxxx
  • 修改为：http://服务器地址:8000/auth?email=用户名%40邮箱域名&authToken=c5618b821d6ccxxxxxxxxxxxx
